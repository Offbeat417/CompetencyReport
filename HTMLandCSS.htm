Hour 1

While HTML is the meat and potatoes, CSS is the onions, carrots and garlic of the soup. It's there to make everything on the web page look pretty, and taste delicious. This particular language has the ability to interlace itself with HTML and design it to your own liking. The beauty of it is that potentially, you can make it look however you want it to.

Also, as a side note, you're actually learning CSS3. Supposedly, it's common to refer to CSS3 as simply CSS, as it is assumed by most individuals would have made the switch. Either way, it's good to make that ascertation when building a resume to impress; ensure you know what you're doing and what you're talking about.

At the end of the day, CSS isn't a lot of code, and it really comes down to personal preference.

In most CSS files created, you'll see HTML tags being used. Essentially, with CSS you can call upon certain parts of an HTML file and tell CSS how we want it to look.

Remember, you MUST end each "rule" put into place in the CSS file with a semicolon(;), otherwise you could cause potentially fatal errors in your code.

Include a link tag in the head section of each document you want to use the style with.
